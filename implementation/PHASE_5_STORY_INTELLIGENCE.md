# Phase 5 — Study Mode Differentiation

## Status: ⬜ TODO

---

## Context

The 4 study characters (Count Cosmo, Dr. Luna, Professor Pip, Arty) exist and work end-to-end — they use the same `StoryScreen` as story mode. The session behaviour and UI are currently identical to story mode. This phase makes the study experience distinct.

---

## 5.1 Study Session Prompt Enhancement

Study characters already have dedicated English-only system prompts that teach through mini-stories. Enhance them to:

- Explicitly prompt the character to **ask the child a question** every 60–90 seconds
- **Wait for a real answer** before continuing
- Give **positive audio feedback**: "Yes! That's exactly right!" / "Ooh good try, let me show you!"
- **Track the concept** being taught within the session ("we're learning addition today")

No backend changes needed — this is prompt engineering only in `backend/characters.py`.

---

## 5.2 Study Screen UI

Replace `StoryScreen` for study sessions with a `StudyScreen` variant:

**Concept panel** (replaces scene image grid):
- Single large image showing the current learning concept
- Examples: a number line, a colour wheel, a letter shape, an animal diagram
- Same image generation pipeline as story mode — just presented differently (one featured image vs scrollable grid)

**End-of-session summary card:**
- Displayed after `sessionState === "ended"`
- Text generated by prompting the model with a final `client_content` message before closing:
  `"Summarise in 1-2 sentences what we learned today in a fun, encouraging way for the child."`
- Displayed as a colourful card: "Today with Count Cosmo you practised: counting to 10, shapes (circle, triangle)"

**Visual theme:**
- Green/teal colour scheme throughout (landing page Study card already uses this)
- Distinguishes study from story visually

## 5.4 End-of-Session Summary Flow

```
User clicks "End Study"
    → useLiveAPI: send final client_content message before closing
        { "client_content": { "parts": [{ "text": "Summarise what we learned today..." }] } }
    → Gemini responds with summary text via outputTranscription
    → turnComplete fires → capture summary text
    → ws.close()
    → StudyScreen shows summary card with captured text
```

No new backend endpoint needed.
