# TaleWeaver — How Story Images Are Generated

There are three image generation moments in TaleWeaver:

1. **Pre-warm** — one image generated before the session starts (`POST /api/story-opening`)
2. **Tool call** — Gemini decides a moment deserves a picture and calls `generate_illustration`
3. **Fallback timer** — if Gemini hasn't triggered an image, one fires on `turnComplete`

Moments 2 and 3 both go through `POST /api/image`. Moment 1 is a separate endpoint.

---

## Pre-warm — `POST /api/story-opening`

Called when the child lands on StoryScreen, before they press Begin.

A single `gemini-2.5-flash-lite` call generates three things in one shot:

```
PLAN:  [one-line story structure: setting | hero situation | 3 beats | resolution]
STORY: [3-4 sentence opening in the character's language]
SCENE: [English painter's description of the opening visual moment]
```

All three are coherent because they come from the same prompt. The model invents the plan first, then writes the opening *from that plan*, then describes the scene matching that opening.

The SCENE description is then fed to the image model to produce the first illustration.

The response returns:
- `opening_text` — what Gemini Live will speak word-for-word at the start
- `story_plan` — compact plan sent to Gemini Live as structural guidance for the session
- `image_data` — base64 first illustration shown immediately when Begin is pressed
- `scene_description` — used as the reference description for the next image (continuity)

### Why a single call?

Previously, plan and opening were generated by two separate parallel requests. The plan could invent "Lily finds a treasure in a crystal cave" while the opening independently invented "Sparkle the phoenix in a jungle". No link between them.

With one call, the model builds the plan and opening as a single coherent thought — the same characters, setting, and story arc flow through all three outputs.

---

During the live story session, images are generated in two ways. Both end at the same
`POST /api/image` endpoint but arrive there via different triggers and with different inputs.

---

## Path A — Gemini calls the tool (preferred)

Gemini Live is given a `generate_illustration` function declaration in its setup message.
When it decides a moment deserves a picture (new scene, character introduction, dramatic
reveal), it emits a **tool call** mid-narration.

### When does Gemini call it?

The system prompt instructs:
> Call this when you describe: a new location, a character appearing for the first time,
> a magical transformation, a dramatic reveal, or any moment that would make a beautiful
> storybook picture. Do NOT call it for dialogue turns, thinking pauses, or routine story
> progression. Call it at most once every 2 story beats.

### What Gemini sends in the tool call

```json
{
  "toolCall": {
    "functionCalls": [{
      "id": "abc123",
      "name": "generate_illustration",
      "args": {
        "scene_description": "Pip the tiny golden hummingbird dives headfirst into a swirling tornado of rainbow candy floss, her wings beating furiously as a laughing cloud rains fizzy lemon drops around her."
      }
    }]
  }
}
```

Gemini writes this in **English** regardless of the story language — the system prompt
explicitly requires it.

### What the frontend does

The frontend **immediately sends a tool response** back so Gemini doesn't pause:

```json
{
  "toolResponse": {
    "functionResponses": [{ "id": "abc123", "response": { "output": "Illustration generated." } }]
  }
}
```

Then it calls `forceImageGeneration(scene_description)` — this **bypasses all rate limits**
since Gemini already chose the right moment.

### What goes to `POST /api/image`

```json
{
  "scene_description": "Pip the tiny golden hummingbird dives headfirst into a swirling tornado of rainbow candy floss, her wings beating furiously as a laughing cloud rains fizzy lemon drops around her.",
  "story_context": "...last 2000 chars of all story speech so far...",
  "image_style": "magical wizard illustration, enchanted library, glowing spell books, starry backdrop, children's fantasy art, warm golden light",
  "session_id": "a1b2c3d4",
  "skip_extraction": true,
  "previous_image_data": "...base64 of last generated image...",
  "previous_image_mime_type": "image/png",
  "previous_scene_description": "A little blue dragon sitting on a cloud eating a jam sandwich."
}
```

`skip_extraction: true` means the backend trusts the description as-is — **no LLM extraction step**.

## Path B — Fallback timer (every N seconds)

If Gemini doesn't call the tool, the frontend fires an image on every `turnComplete` event,
subject to rate limiting.

### When does `turnComplete` fire?

Every time Gemini finishes speaking a full turn. The frontend has been accumulating the
output transcription chunks into `outputTextAccRef`. When `turnComplete` arrives, that
accumulated text is the full narration Gemini just spoke.

### Rate limiting rules (in `useStoryImages.ts`)

```
1. Wait 8 seconds after session start before generating any image
2. Minimum interval between images (default 8s, user-adjustable 5–60s)
3. If Vertex AI returns 429 → silently discard, reset timer so next turn retries
```

### What the frontend sends as the transcription

Gemini just narrated (in Tamil, for Hanuman):

> கடலின் ஆழத்தில், ஒரு சிறிய ஆமை வேகமாக நீந்தியது. அதன் ஓட்டில் பளபளக்கும் நட்சத்திரங்கள் இருந்தன!
> ("Deep in the ocean, a tiny turtle swam swiftly. On its shell were glittering stars!")

This **raw Tamil text** goes into `scene_description`.

The rolling story context (last ~2000 chars of all prior speech) also goes in as `story_context`.

### What goes to `POST /api/image`

```json
{
  "scene_description": "கடலின் ஆழத்தில், ஒரு சிறிய ஆமை வேகமாக நீந்தியது...",
  "story_context": "...earlier Tamil narration, last 2000 chars...",
  "image_style": "bright Indian mythology illustration, lush jungle, lotus flowers, warm golden light, children's storybook art, Tamil style",
  "session_id": "a1b2c3d4",
  "skip_extraction": false,
  "previous_image_data": "...base64...",
  "previous_image_mime_type": "image/png",
  "previous_scene_description": "A small turtle with star-patterned shell swimming in a coral reef."
}
```

`skip_extraction: false` — the backend must extract a visual description first.

## Inside `POST /api/image` — what happens step by step

### Step 1 — Extract a visual scene (Path B only)

**Model:** `gemini-2.5-flash-lite` (via Vertex AI, `us-central1`)

The raw narration (may be in any language) goes to Flash Lite with this prompt:

```
Story so far (use this to identify the exact characters, species, names, and setting):
...last 1500 chars of story_context...

The storyteller just spoke the narration below.
Identify the single most visually interesting moment in this narration and describe it
as a concise English image description (2-3 sentences).
Be SPECIFIC — use the EXACT character type from the story context above (e.g. 'rubber chicken toy',
'wooden puppet', 'friendly bear cub').
NEVER replace a specific character type with a vague word — FORBIDDEN words:
'creature', 'character', 'figure', 'being', 'entity', 'animal' (unless that IS the type).
Name the exact setting (forest, teacup ride, ocean, etc.) and the precise action happening.
CRITICAL: Describe only story characters in their story world.
NEVER write 'a child holds...', 'a person holds...', or 'someone is holding...'.
No dialogue, no abstract concepts — purely visual.

Narration: கடலின் ஆழத்தில், ஒரு சிறிய ஆமை வேகமாக நீந்தியது...
```

Flash Lite outputs something like:

> A tiny turtle with a shell covered in glittering gold star patterns swims through a deep
> underwater coral canyon, bioluminescent sea anemones swaying around it, shafts of teal
> light filtering down from the surface far above.

**Path A skips this step entirely** — Gemini's own description is used directly.

---

### Step 2 — Build the full image prompt

The extracted (or tool-provided) scene is combined with a safety prefix and the character's art style:

```
child-safe illustration, age-appropriate for children aged 4-10, no violence, no scary content,
no adult themes, cartoon style, no text, no words, no letters, no captions, no speech bubbles,
no labels, purely visual, [character art style], [scene description]
```

Concrete example for Hanuman + the turtle scene:

```
child-safe illustration, age-appropriate for children aged 4-10, no violence, no scary content,
no adult themes, cartoon style, no text, no words, no letters, no captions, no speech bubbles,
no labels, purely visual, bright Indian mythology illustration, lush jungle, lotus flowers,
warm golden light, children's storybook art, Tamil style, A tiny turtle with a shell covered
in glittering gold star patterns swims through a deep underwater coral canyon, bioluminescent
sea anemones swaying around it, shafts of teal light filtering down from the surface far above.
```

---

### Step 3 — Generate the image

**Model:** set by `IMAGE_MODEL` env var (currently `gemini-3.1-flash-image-preview`)

**Client:** `_api_key_client` (Gemini API key via AI Studio) if set, else `_client` (Vertex AI).
API Studio is preferred because it has shorter rate-limit intervals than Vertex.

If a previous image exists, it is passed alongside the prompt as a reference for visual continuity:

```
[text: continuity instructions explaining when to reuse vs rebuild the background]
[inline_data: previous scene image (base64 PNG/JPEG)]
[text: "New scene to illustrate: child-safe illustration... A tiny turtle..."]
```

The model generates a new image that:
- Matches the new scene description
- Keeps any carry-over characters visually consistent with the previous image
- Rebuilds the background if the setting has changed

## Summary — full example end to end

```
Gemini Live narrates (Tamil):
  "கடலின் ஆழத்தில், ஒரு சிறிய ஆமை வேகமாக நீந்தியது. அதன் ஓட்டில் பளபளக்கும் நட்சத்திரங்கள்!"

turnComplete fires
  → storyContext updated (append + keep last 2000 chars)
  → rate limit check passes (8s session delay, 8s interval)
  → POST /api/image {scene_description: <Tamil text>, skip_extraction: false, ...}

Backend Step 1 — gemini-2.5-flash-lite extracts:
  "A tiny turtle with a gold star-patterned shell swims through a teal coral canyon..."

Backend Step 2 — build prompt:
  "child-safe illustration... Tamil style... A tiny turtle with a gold star-patterned shell..."

Backend Step 3 — gemini-3.1-flash-image-preview generates image
  → returns base64 PNG

Frontend:
  → scene card flips from "loading" to the generated image
  → lastImageRef updated (used as reference for the next image)
```

## Two-path comparison

| | Path A (tool call) | Path B (fallback timer) |
|---|---|---|
| Trigger | Gemini decides | `turnComplete` + rate limit |
| Who writes the scene description | Gemini (clean English) | Frontend sends raw transcription |
| Extraction step | Skipped | `gemini-2.5-flash-lite` runs |
| Rate limit bypass | Yes | No (8s session delay, min interval) |
| Quality | Higher (Gemini chose the moment) | Variable (fires on a timer) |
| Image model | `gemini-3.1-flash-image-preview` | `gemini-3.1-flash-image-preview` |
